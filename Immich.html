Immich Self-Hosting Guide

Take control of your memories. This guide will walk you through setting up Immich, a powerful self-hosted photo and video backup solution, on a dedicated server.

Important Note: This guide assumes you have a dedicated server (like an Intel N100 mini PC, Raspberry Pi with external storage, or a home server) running Docker. The Flint 2 router does NOT support Docker, so a separate machine is required for Immich.

What You'll Need

  • A dedicated server with Docker and Docker Compose installed.
  • Basic command-line knowledge.
  • A reverse proxy (e.g., Nginx Proxy Manager, Caddy) if you want external access (recommended).
  • A domain name (optional, but recommended for easy external access).

Step-by-Step Installation

Step 1: Prepare Your Server

Update and Install Dependencies

Ensure your server's package list is up-to-date and install Git (if not already present):

sudo apt update
sudo apt upgrade -y
sudo apt install git -y
Step 2: Clone Immich Repository

Download Immich Docker Compose Files

Navigate to your desired installation directory and clone the Immich example repository:

cd /opt # Or any directory where you want to store Immich files
git clone https://github.com/immich-app/immich-docs.git
cd immich-docs/install

This will give you the necessary docker-compose.yml and other configuration files.

Cloning the Immich repository on your server.

Step 3: Configure Environment Variables

Set Up Your .env File

Copy the example environment file and then edit it to set your desired passwords and other settings. You'll primarily want to focus on DB_PASSWORD and potentially port mappings.

cp .env.example .env
nano .env # Or use your preferred text editor like vim

Key variables to review:

  • DB_PASSWORD=your_secure_password (CRITICAL: Change this!)
  • UPLOAD_LOCATION=/usr/src/app/upload (This is the default path inside the container. You'll map a host volume to this later.)
Step 4: Create Data Directories

Persistent Storage for Immich

Immich needs persistent storage for your uploaded photos/videos and for its database. Create these directories on your host machine and ensure Docker has permission to write to them.

sudo mkdir -p /mnt/immich/upload # For your photos/videos
sudo mkdir -p /mnt/immich/database # For PostgreSQL data
sudo chown -R 1000:1000 /mnt/immich # Give ownership to the Immich user ID (default)

Note: Adjust /mnt/immich to your preferred storage location on your server.

Step 5: Edit Docker Compose File

Map Volumes and Configure Ports

Open the docker-compose.yml file and modify the volumes section for the immich-server, immich-microservices, and database services to point to your newly created host directories. Also, ensure your ports are correctly mapped.

nano docker-compose.yml

Example volume mappings (look for these sections and adjust):

For immich-server and immich-microservices:

volumes:
      - /mnt/immich/upload:/usr/src/app/upload

For database:

volumes:
      - /mnt/immich/database:/var/lib/postgresql/data

Port Mappings: Ensure the `ports` section for `immich-web` exposes a port for the web interface (default is 3000:80, meaning host port 3000 to container port 80). Adjust if needed, e.g., 8080:80.

Modifying the docker-compose.yml file to map persistent volumes.

Step 6: Start Immich

Launch Docker Containers

Once your .env and docker-compose.yml files are configured, start Immich:

docker compose up -d

The -d flag runs the containers in detached mode, meaning they will run in the background. It might take a few minutes for all services to start up completely.

Running docker compose up -d to launch Immich services.

Step 7: Access Immich Web UI

Initial Setup

Open your web browser and navigate to your server's IP address and the mapped port (e.g., http://your-server-ip:3000). You should be greeted by the Immich setup screen.

Create your admin account. This will be the first user in Immich.

Creating your first Immich admin account.

Step 8: Set Up Reverse Proxy (Optional, Recommended for External Access)

Secure External Access with Nginx Proxy Manager

If you plan to access Immich outside your home network or use a custom domain, a reverse proxy is highly recommended for SSL encryption and easier access.

Using Nginx Proxy Manager (NPM):

  1. In NPM, create a new Proxy Host.
  2. Domain Names: Enter your domain (e.g., photos.yourdomain.com).
  3. Scheme: http
  4. Forward Hostname / IP: Enter your Immich server's local IP address.
  5. Forward Port: Enter the port Immich's web UI is running on (e.g., 3000).
  6. Go to the SSL tab, request a new SSL certificate, and enable "Force SSL" and "HSTS."
  7. Save the configuration.

Configuring a reverse proxy for Immich with Nginx Proxy Manager.

Step 9: Mobile App Setup

Backup Photos from Your Phone

Download the Immich mobile app for Android or iOS.

  1. Open the app and enter your Immich server URL (e.g., https://photos.yourdomain.com if using a reverse proxy, or http://your-server-ip:3000 if local).
  2. Log in with the admin credentials you created.
  3. Configure your backup settings (e.g., Wi-Fi only, charging only).

The Immich mobile app for seamless photo and video backup.

Maintenance Commands

  • Stop Immich: docker compose down
  • Start Immich: docker compose up -d
  • Update Immich: 
    cd /opt/immich-docs/install # Your Immich installation directory
git pull
docker compose pull
docker compose up -d
  • View Logs: docker compose logs -f

Further Customization & Tips

  • ExifTool: Immich leverages ExifTool for metadata. Ensure your server has it installed for best performance.
  • GPU Transcoding: If your server has a compatible GPU, configure Immich to use it for faster video transcoding. Refer to the official Immich documentation.
  • Multiple Users: Create additional users directly from the Immich web interface. Each user gets their own private library.
  • External Libraries: Connect existing photo/video folders to Immich for easy viewing and management without re-uploading.
